## 二、功能需求

### 2.1 实时视频分析（P0）

#### 功能描述
支持用户上传视频或实时拍摄，进行连续帧姿态分析，提供实时反馈。

#### 用户故事
> 作为用户，我希望能够录制我的打球视频并实时获得分析，这样我可以立即了解动作问题并调整。

#### 功能细节

**视频上传**:
- 支持格式：MP4、MOV
- 最大时长：60 秒
- 最大文件大小：50MB
- 推荐分辨率：720p/1080p

**实时分析**:
- 帧率：30fps
- 分析延迟：< 500ms
- 输出：每帧关键点 + 每 5 秒汇总报告

**分析结果**:
- 整体评分（0-100）
- 关键问题列表（按时间戳）
- 最佳帧截图
- 改进建议

#### 技术方案
- 前端：WebRTC + MediaPipe Pose（客户端分析）
- 后端：视频分帧 + 批量分析（服务端备用）
- 降级方案：客户端分析失败时，上传到服务端分析

#### 验收标准
- [ ] 视频上传成功
- [ ] 实时分析延迟 < 500ms
- [ ] 分析结果准确率 > 85%
- [ ] 支持 720p/30fps 流畅分析

### 2.2 标准动作库（P0）

#### 功能描述
提供职业选手标准动作视频和骨架数据，供用户对比学习。

#### 用户故事
> 作为用户，我希望看到职业选手的标准动作，这样可以知道正确的动作应该是什么样的。

#### 功能细节

**动作分类**:

| 类别 | 动作 | 数量 |
|------|------|------|
| 正手 | 正手攻球（初/中/高） | 3 |
| 反手 | 反手推挡（初/中/高） | 3 |
| 发球 | 正手发球、反手发球、侧旋发球、下旋发球、逆旋转发球 | 5 |
| **合计** | | **11** |

**每个动作包含**:
- 职业选手演示视频（多角度）
- 骨架关键点数据
- 关键角度范围（肘部、膝部、躯干）
- 常见错误说明
- 练习方法

#### 数据来源
- 职业选手动作采集（合作球馆/教练）
- 公开比赛视频（标注后使用）
- 3D 动画模拟（备选）

#### 验收标准
- [ ] 11 个标准动作全部上线
- [ ] 每个动作至少 2 个角度视频
- [ ] 骨架数据准确
- [ ] 支持动作搜索和筛选

### 2.3 动作对比可视化（P0）

#### 功能描述
将用户动作与标准动作并排对比，直观展示差异。

#### 用户故事
> 作为用户，我希望能够直观地看到我的动作和标准动作的差异，这样我知道具体哪里需要改进。

#### 功能细节

**对比模式**:

1. **并排对比**
   - 左侧：用户动作视频
   - 右侧：标准动作视频
   - 同步播放

2. **骨架叠加**
   - 半透明叠加显示
   - 用户骨架（蓝色）
   - 标准骨架（绿色）
   - 差异点高亮（红色）

3. **关键角度对比**
   - 肘部角度：用户 85° vs 标准 95°
   - 膝部角度：用户 130° vs 标准 120°
   - 躯干角度：用户 20° vs 标准 10°

**交互功能**:
- 播放/暂停
- 慢放（0.5x）
- 逐帧前进/后退
- 关键帧标记

#### 技术方案
- 前端：Canvas/SVG 绘制骨架
- 视频同步：时间戳对齐
- 性能优化：WebGL 加速

#### 验收标准
- [ ] 并排对比流畅
- [ ] 骨架叠加准确
- [ ] 角度数值实时显示
- [ ] 支持慢放和逐帧

### 2.4 训练计划（P1）

#### 功能描述
根据用户动作分析结果，生成个性化训练计划。

#### 用户故事
> 作为用户，我希望有一个针对性的训练计划，这样我知道每天应该练习什么。

#### 功能细节

**计划生成**:
- 基于动作问题（最严重的 3 个问题）
- 基于用户水平（初/中/高）
- 基于可用时间（15/30/60 分钟）

**计划内容**:
- 每日训练任务（3-5 个动作）
- 每个动作练习次数/时长
- 视频教程链接
- 预期见效时间

**进度追踪**:
- 每日打卡
- 完成度统计
- 进步曲线

#### 验收标准
- [ ] 自动生成训练计划
- [ ] 计划个性化
- [ ] 支持打卡和追踪
- [ ] 会员专属功能
