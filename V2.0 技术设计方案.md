## 二、核心功能设计

### 2.1 实时视频分析

#### 方案 A: 客户端分析（推荐）

**流程**:
```
用户录制视频 → 前端 MediaPipe 分析 → 发送结果到后端 → 保存记录
```

**优点**:
- ✅ 低延迟（<100ms）
- ✅ 服务器负载低
- ✅ 离线可用

**缺点**:
- ❌ 依赖客户端性能
- ❌ 模型版本难控制

**实现**:
```javascript
// 前端使用 MediaPipe Pose
import { Pose } from '@mediapipe/pose'

const pose = new Pose({
  locateFile: (file) => {
    return `https://cdn.jsdelivr.net/npm/@mediapipe/pose/${file}`
  }
})

pose.onResults(onPoseResults)
await pose.send({ image: videoElement })
```

#### 方案 B: 服务端分析

**流程**:
```
用户上传视频 → 后端分帧 → MediaPipe 分析 → 返回结果
```

**优点**:
- ✅ 模型版本统一
- ✅ 结果一致性高

**缺点**:
- ❌ 延迟高（2-5 秒）
- ❌ 服务器负载高

**决策**: MVP 阶段采用**方案 A（客户端分析）**

### 2.3 动作对比可视化

#### 对比模式

**1. 并排对比**
```
┌─────────────┬─────────────┐
│  用户动作   │  标准动作   │
│   (视频)    │   (视频)    │
│             │             │
│  分数：85   │  分数：100  │
└─────────────┴─────────────┘
```

**2. 骨架叠加**
```
┌─────────────────────────┐
│    用户 (蓝色)          │
│    +                    │
│    标准 (绿色半透明)     │
│                         │
│    差异点 (红色高亮)     │
└─────────────────────────┘
```

**3. 关键角度对比**
```
┌─────────────────────────┐
│ 肘部角度                │
│ 用户：61°  标准：95°    │
│ ████████░░░░░░░░ 61%    │
│                         │
│ 膝部角度                │
│ 用户：35°  标准：120°   │
│ ███░░░░░░░░░░░░░ 29%    │
└─────────────────────────┘
```

#### 技术实现

```javascript
// Canvas 绘制骨架
function drawSkeleton(ctx, keypoints, color) {
  const connections = [
    [11, 12], // 肩膀
    [11, 13], [13, 15], // 左臂
    [12, 14], [14, 16], // 右臂
    [23, 24], // 臀部
    [23, 25], [25, 27], // 左腿
    [24, 26], [26, 28]  // 右腿
  ]
  
  connections.forEach(([i, j]) => {
    ctx.beginPath()
    ctx.moveTo(keypoints[i].x, keypoints[i].y)
    ctx.lineTo(keypoints[j].x, keypoints[j].y)
    ctx.strokeStyle = color
    ctx.lineWidth = 3
    ctx.stroke()
  })
}
```

## 五、性能优化

- **视频压缩**: 上传前压缩至 720p

- **文件清理**: 24 小时自动删除上传文件
- **结果缓存**: 相同视频不重复分析
- **队列处理**: 高峰期排队提示
